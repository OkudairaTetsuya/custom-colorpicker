1. 開発目的・背景
目的: ユーザーがスマホケースのオリジナルカラーを自由に選択し、完成イメージをリアルタイムで確認できる「シミュレーター」を提供することで、購買体験の質を向上させ、コンバージョン率（CVR）を高める。

ターゲット: スマホユーザー中心。低スペック端末や低速回線環境でもストレスなく動作することを目指す。

2. 実装詳細仕様書
2.1 機能要件
カラー選択エリア（2Dマップ）: 彩度（Saturation）と明度（Value）を直感的に選択。

色相スライダー（1Dバー）: 色相（Hue）を0〜360度で変更。

数値表示: 選択中の色のHEX値を表示。

リアルタイムシミュレーション: 選択した色をスマホケースの「背面レイヤー」に即時反映。

2.2 技術スタック
HTML5 / CSS3: レイアウトおよびグラデーション表現。

Vanilla JavaScript: 色変換ロジックおよびイベント処理（ライブラリ使用禁止）。

Asset: 透過PNG画像2枚（後述）。

2.3 プレビューの構造（低負荷実装）
3層のレイヤーを重ねる方式を採用。

Base Layer (最背面): JavaScriptで背景色（backgroundColor）を変更するだけの div。

Texture Layer (中間): スマホケースの「影」と「光沢」だけを描いた透明PNG。CSSの mix-blend-mode: multiply（またはoverlay）を使用。

Frame Layer (最前面): カメラ穴やロゴなど、色が変わらない部分の透明PNG。

3. 制作指示書（AIエージェントへのプロンプト）
以下の指示をAIエージェントに入力してください。

Role: シニア・フロントエンドエンジニア
Task: 楽天GOLDに配置する「スマホケース・カラーシミュレーター」を作成してください。

制約条件:

外部ライブラリ（jQuery, React等）や外部API、Webフォントは一切使用しない「バニラJS」で記述すること。

楽天GOLD（静的サーバー）で動作するよう、HTML/CSS/JSを完結させること。

低スペックスマホでも滑らかに動くよう、複雑な計算やCanvasの多用を避け、CSSグラデーションとDOM操作を中心に構成すること。

画像内の「Googleカラーピッカー」のUIを模倣し、その下に「スマホケースのプレビュー」を表示するレイアウトにすること。

実装詳細:

Color Logic: HSV色空間をベースに計算し、HEX値に変換して出力すること。

UI Components:

彩度・明度を選択する正方形のエリア（背景は選択中のHue）。

虹色の色相選択スライダー（input type="range" をカスタムするか、divで作成）。

Preview:

#case-preview 要素の中に、背景色用レイヤーと、質感用画像（texture.png）、外枠用画像（frame.png）を重ねる構造にすること。

Events:

PCのクリックだけでなく、スマホのタッチイベント（touchstart, touchmove）でもスムーズに色が追従すること。

Output:

index.html, style.css, script.js の3つのコードを出力してください。

【注意点・例外】
画像素材の命名: 上記指示書では texture.png と frame.png という名前にしています。実際に画像を作成する際は、この名前に合わせるか、コード内のパスを書き換える必要があります。

色の誤差: ディスプレイの特性上、シミュレーションの色と実際の印刷色は必ずズレます。「※画面上の色はイメージです」という一文を必ずUI付近に入れてください。
